# Pacman Repository
Repository for custom packages to be managed by the pacman package manager.

You might ask, why create a pacman repository when you could simply upload it to the AUR so each user can compile it themselves, instead of creating tags with versions based on their computer?

I plan to install Linux on some friends' devices who have never used it before, so I want to keep it ready and easy for them to install and update. For example, I want to use the same package names, so if I think a package used on multiple computers needs a modification, I can upload the modified package to this repository, and users will only need to update their systems to install it.

I think this will also help me better understand how pacman packages work and how CachyOS compiles its packages based on the CPU.

I also thought it would be cool if running pacman displayed "estoyburridow" as a package database.
